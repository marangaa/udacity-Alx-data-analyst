gathering and assessing data

the data to work on was given from the following files: - a dataset containing archived twitter data from the weRateDogs website twitter-archive-enhanced.csv` - the tweet image predictions file, obtained from running images through a neural network and obtained programmatically - additional data to be obtained from the Twitter API, using the tweepy library and twitter developer credentials once you apply for a developer account the project required storing the data from the API into a file in json format, working with the Twitter API through tweepy a python library was fairly easy. the instructors mentioned the data would take some time to be collected, saved us alot of time from thinking some code did not work.

assessing the data

data assessment was done in two stages, visual assessment and programmatic, when assessing data visually, the source column which had the platform names still had href tags which were not useful and actual content needed to be extracted the dog names have issues, ie some names are clearly not names, some have ‘none’ instead of NAn, some were a single letter which indicates the dog probably had two names and was truncated to one while obtaining the data also dog stages were spread across different columns. There were alot of retweets in the dataset, which were not needed for analysis and needed to be removed from the dataset accessing the data programmatically allowed us to identify some issues such - incorrect data types for the all the timestamp columns - ID columns in the dataset are of int type - some tweets don't contain useful data for use in the project

cleaning the data
